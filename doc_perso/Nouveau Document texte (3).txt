Bonne idée. Si tu veux un banc de test crédible pour l’entreprise, il faut surtout un scoring automatique simple mais robuste. Le but n’est pas de juger “parfaitement” les réponses, mais de comparer des pipelines, des modèles et des coûts.

Voici une approche qui marche bien en pratique.

1️⃣ Structure d’un test

Chaque question test doit avoir :

{
  "question": "...",
  "expected_points": [
    "point clé 1",
    "point clé 2",
    "point clé 3"
  ],
  "type": "analyse_document",
  "difficulty": "medium"
}

Ainsi ton système peut vérifier si la réponse contient les éléments attendus.

2️⃣ Scoring simple (3 critères)

Tu peux calculer un score global :

score = pertinence + complétude + hallucination
pertinence

Est-ce que la réponse traite bien la question ?

0 → hors sujet
1 → partiel
2 → bon

complétude

Combien de points attendus sont couverts ?

points_couverts / points_attendus
hallucination

Un LLM critique peut analyser :

0 → aucune hallucination
1 → doute
2 → hallucination claire
3️⃣ Évaluateur LLM

Tu peux utiliser un modèle peu cher pour l’évaluation.

Exemple prompt :

Analyse la réponse suivante.

Question : ...
Réponse : ...

Points attendus :
- ...
- ...

Évalue :
1 pertinence (0-2)
1 complétude (0-2)
1 hallucination (0-2)

Retourne JSON.

Donc tu obtiens :

{
  "pertinence": 2,
  "completude": 1,
  "hallucination": 0
}
4️⃣ Score final

Exemple :

score = pertinence*0.4 + completude*0.4 + (2-hallucination)*0.2

Résultat :

score 0 → 1
